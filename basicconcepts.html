Internet Concepts

Image: https://blog.nxp.com

Overview

Say you wanted to grab a webpage from a server. If we were to watch your request, it would most likely go from your computer to your wireless access point. From the access point it would go to your router which would point it to your Internet Service Provider or ISP. From there your website request would be looked up at a Domain Name Server (DNS) which simply connects the name of the webpage with an internet address. Your request would then go to this internet address, possibly traversing thousands of miles of land or sea before the web page receives your request. It would then send a response which your computer would get use to show the webpage that you requested.

Routing

In a network like the internet, different autonomous systems make up the nodes of the network. Different network structures have different ways to transmit information from one node to other nodes. Routing is the process to select the best way for delivering data and information. Routing directs information from its source to its final destination via intermediate nodes. Also, intermediate nodes can store different routes in memory. Each single element can choose the best way to be transferred. For example, we have to transfer information from node 1 to node 6. There are lots of different ways to deliver the information. The best way could be 1 -> 8 -> 7 -> 6. The process of selecting this path is called routing.

OSI Model

The OSI (Open Systems Interconnection) model is a model that describes the complex process of how data is transferred between computers. As you can see the data starts off at the application layer where it is then deconstructed into smaller and smaller sections eventually reaching the transport layer. The data is then split into smaller sections called packets which traverse the internet. These packets are transmitted be being sent through cables, sometimes under oceans, and then reconstructed by your destination into the data you wished to send. The OSI model describes this process and it splits the process of transferring data into seven manageable layers and describes how each layer contributes to the overall process of sending and receiving data. It is often shown visually in order to help people remember the layers and the order in which they occur when transmitting and receiving data.

TCP and UDP

Data on the internet is transferred in the form of packets and how your computer interprets these packets is based on which protocol you use (TCP or UDP). In general TCP or Transmission Control Protocol requires all packets be received at the end, kind of like a checklist, and if a packet is missing it will request that individual packet. This is useful for when you don’t want missing or dropped frames when you are watching a movie. The other protocol is UDP or User Datagram Protocol which allows dropped packets, which is useful for streaming services like Twitch, where it doesn’t really matter if you miss a couple of frames. The most common is TCP.

3-Way Handshake

Creating a connection between a computer and server is difficult. In order to combat this, the 3 way handshake was developed and is used today. First your computer sends a SYN packet to the server, asking it to develop a connection. However, after sending the packet, the computer has no idea if the server received the data. Therefore the server sends back a SYN/ACK packet which says that it did receive your request and that you can create a connection. However the server does not know if the computer received the response and thus has to wait for the computer to send an ACK packet before connecting with it. This is to ensure that the computer is ready to connect. If the SYN/ACK packet was not delivered to the computer, then the server would just be waiting with an open connection for the computer to connect.

Jargon Buster

  • Domain Name Server: A server that is able to convert a domain name (www.example.com) into an IP address (xxx.xxx.xx.xx) so that computers on the network can identify each other.
  • Autonomous System: A collection of computers under the authority of a particular network administrator, such as your ISP.
