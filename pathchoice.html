History

In the 1960s work was carried out on the 'ARPAnet'. This was the predecessor to the internet and was an experimental network to share resources between universities, as a solution to a lack of funds. In 1969, the first message was sent from UCLA to Stanford and simply said 'lo', but it was a start. UCL was the first place outside of the US to have a connection to the ARPAnet.

The internet became commercially available in the 1990s, and this was exciting because it was a new way for people to share information. However, in recent years the internet has obviously become very crowded, and as a result the internet slows down during congestion. Current research aims to solve this problem.

TCP is the protocol (set procedure) currently used to control the sending and receiving of packets over the internet. It was invented by Bob Kahn and Vint Cerf in the 1970s. Current Issues

  • Scalability: The global Border Gateway Protocol (BGP) is not in an optimal state. This table is filled with IP address prefixes, and some of them are more specific than others. For example, 125.68.45 is more specific than 125.68. Around half the entries in the BGP table are these more specific prefixes. In a sense, this data could be considered redundant because the less specific prefix should suffice for routing. ISPs are concerned about the rate at which the BGP is growing.
  • Slow failure recovery: The BGP does not update immediately whenever there are network failures. This means that packets may be routed along the broken path for a small time frame when the network first initially goes down. This can disrupt the user experience.

Goals and Focus

UCL have proposed using a multipath-capable Transmission Control Protocol (TCP). This would work by having the sender and receiver of data packets both multihomed. This means that they would have multiple IP addresses. When they first interact, these IP addresses could all be exchanged, allowing multiple paths to be set up. One advantage of this is that if a packet is dropped, the system can respond more quickly and resend the packet on another path.

UCL have also suggested that congestion should be controlled by Explicit Congestion Notifications (ECN) markers. This would allow the transmission rate to incrementally steadily, and once a ECN is sent, or packet loss is detected, the transmission rate could be cut.

Jargon Buster

  • TCP: Transmission Control Protocol. A set of rules that control how information is set over the internet.
  • BGP: Border Gateway Protocol. This is the internet's way of routing traffic through the internet by connecting autonomous systems. The idea is pick the shortest path.
  • Autonomous System: This is a network of computers that make up the network of the internet. This is why we refered to the internet as a "network of networks." These sub-networks could be part of your ISP for example.
  • ECN: Explicit Congestion Notifications. These are markers in a packet that simply indicate congestion on the path.
